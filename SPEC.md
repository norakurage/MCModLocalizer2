# MCModLocalizer2 仕様書

## 1. プロジェクト概要

### 目的
Minecraft の Mod JAR ファイルから英語ローカライゼーションファイル (`en_us.json`) を抽出し、Google Gemini API で日本語化 (`ja_jp.json`) してリソースパックとして出力する GUI アプリケーション。

### スコープ
- Windows (x64) / macOS (arm64, x64) 対応のデスクトップアプリ
- Python/Flet 版 `MCModLocalizer` の完全 Electron 再実装
- インストーラー形式で配布 (.exe / .dmg)
- 自動アップデート対応

### 対象ユーザー
Minecraft の Mod を日本語でプレイしたい個人ユーザー。Python 環境不要でそのまま使える。

## 2. 機能要件

### 2.1 翻訳・リソースパック生成 (メイン機能)

#### 入力
- `mods/` フォルダを指定する (フォルダ選択ダイアログ または D&D)
- 出力フォルダを指定する (デフォルト: 後述の自動推測)
- 個別 JAR ファイルの D&D にも対応

#### 出力フォルダの自動推測
- 入力が `<root>/mods/` の場合、出力を `<root>/resourcepacks/<root_name>_localize/` と推測
- 推測したパスを UI に表示し、ユーザーが確認・変更できる
- 変更した場合は electron-store に記憶

#### 処理フロー
1. `mods/` 配下の `.jar` を列挙
2. 各 JAR から `assets/<modid>/lang/en_us.json` を抽出 (JAR 内 JAR も再帰スキャン)
3. 既存の `ja_jp.json` があれば読み込み、未翻訳エントリのみを対象にする
4. テキストをチャンクに分割して Gemini API へ順次送信
5. 翻訳結果を復元し `ja_jp.json` として出力
6. `pack.mcmeta` と `pack.png` を生成してリソースパックを完成させる

#### 差分翻訳
- 出力先に既存の `ja_jp.json` がある場合、値が空文字のキーのみを翻訳対象とする
- 翻訳済みエントリは上書きしない

#### 複数 Mod の並列処理
- ユーザーが並列数 N を設定可能 (デフォルト: 1、範囲: 1〜8)
- N 個の Mod を同時に処理する

#### 中断・再開
- **即時停止**: 「停止」ボタン押下で進行中の Gemini API コールを `AbortController` でキャンセル
- **中間ファイル保存**: 停止時またはエラー時に `<out>/.resume/<modid>.json` へ翻訳済み分を保存
- 次回起動時に中間ファイルが存在すれば「再開」ボタンを提示

### 2.2 トークン使用状況

- 今セッションの使用トークン数 (入力/出力/合計)
- 概算コスト (USD)
- 過去の実行履歴 (日時・モデル・トークン数・コスト) を表形式で表示
- 履歴は electron-store に永続化

### 2.3 設定

| 設定項目 | 型 | デフォルト値 | 説明 |
|---|---|---|---|
| `model` | string | `gemini-2.5-flash` | 使用する Gemini モデル |
| `parallelism` | number | 1 | 並列処理数 (1〜8) |
| `chunkMaxChars` | number | 6000 | バッチあたりの最大文字数 |
| `chunkMaxItems` | number | 80 | バッチあたりの最大エントリ数 |
| `sleepInterval` | number | 400 | バッチ間スリープ (ms) |
| `theme` | `light` \| `dark` \| `system` | `system` | UI テーマ |

#### API キー管理
- 設定タブの「API キー」欄に入力し「保存」→ OS Keychain (keytar) に格納
- マスク表示 (`••••••••••`)
- 「削除」ボタンで Keychain から消去
- アプリ初期化機能: API キー・利用履歴・設定をすべてリセット

### 2.4 リソースパック生成仕様

```
<出力フォルダ>/
├── pack.mcmeta      # description: "Generated by MCModLocalizer"
├── pack.png         # アプリ内蔵アイコン (assets/icon.png をコピー)
└── assets/
    └── <modid>/
        └── lang/
            └── ja_jp.json
```

`pack.mcmeta` の内容:
```json
{
  "pack": {
    "pack_format": 34,
    "description": "Generated by MCModLocalizer"
  }
}
```

## 5. コアロジック詳細

### 5.1 JAR スキャン (`jarReader.ts`)

```
scanJar(jarPath):
  zipを開く
  全エントリを列挙
    - assets/<modid>/lang/en_us.json → 抽出
    - *.jar → 再帰的に scanJar を呼ぶ (深さ制限: 2)
  各 modid の en_us.json を返す
  複数 modid がある場合、エントリ数が最多のものを採用
```

### 5.2 トークン保護 (`tokenProtection.ts`)

保護対象パターン (Python 版 `constants.py` から移植):
```
%1$s, %2$d  → %\d+\$[sd]
%s, %d      → %[sd]
{name}      → \{[a-zA-Z0-9_]+\}
{0}         → \{\d+\}
§a 等       → §[0-9a-fk-or]
\n, \t, \r  → エスケープシーケンス
```
各マッチを `‹T0›`, `‹T1›`, ... に置換して保護し、翻訳後に復元する。

### 5.3 チャンク分割 (`chunking.ts`)

```typescript
function chunkPairs(
  pairs: [string, string][],
  maxChars: number,   // デフォルト 6000
  maxItems: number,   // デフォルト 80
): [string, string][][]
```
JSON エンコード後の文字数で計算。どちらかの上限を超えたら新しいチャンクを開始。

### 5.4 Gemini API 呼び出し (`geminiClient.ts`)

- エンドポイント: `https://generativelanguage.googleapis.com/v1beta/openai/chat/completions`
- 認証: `Authorization: Bearer <api_key>`
- レスポンス形式を JSON Schema で強制: `{"items": ["訳1", "訳2", ...]}`
- `AbortController` のシグナルを fetch に渡し、即時キャンセルに対応
- **リトライ戦略**:
  - HTTP 429/500/502/503 → Exponential Backoff (初期 5s、最大 5 回)
  - HTTP 400/401/403 → 即時失敗
  - ネットワークエラー → Exponential Backoff
- 翻訳結果が期待数に満たない場合:
  1. 「配列要素数を入力と同じにすること」を追記して再試行 (1 回)
  2. それでも不足なら不足分のみ再帰的に再翻訳 (最大深さ 2)

### 5.5 翻訳ワークフロー (`translation.ts`)

```
for each mod (N 並列):
  1. en_us.json をロード
  2. 既存 ja_jp.json をロード (差分翻訳の基準)
  3. 中間ファイル (.resume/<modid>.json) があれば読み込み
  4. 未翻訳キーを抽出
  5. トークン保護を適用
  6. チャンク分割
  7. チャンクごとに Gemini API 呼び出し
     → ログ・進捗を IPC でプッシュ
     → 停止シグナルを確認
  8. トークンを復元
  9. ja_jp.json を書き出し
  10. 成功なら中間ファイルを削除
```

## 7. エラーハンドリング

| エラー種別 | 挙動 |
|---|---|
| API キー未設定 | 「翻訳を開始」ボタンを無効化、設定タブに誘導メッセージ |
| HTTP 401/403 | ログに赤字エラー、処理停止、API キー確認を促す |
| HTTP 429 | ログに警告、Exponential Backoff でリトライ (最大 5 回) |
| HTTP 500/502/503 | 同上 |
| ネットワーク切断 | 同上 |
| リトライ上限超過 | 中間ファイルを保存、ログにエラー、処理停止 |
| JAR が壊れている | そのファイルをスキップ、ログに警告 |
| 翻訳数不足 | 再試行→サブセット再翻訳→それでも失敗なら RuntimeError |
| 即時停止 | AbortError を受けて中間ファイルを保存 |

## 9. 既知の制約・トレードオフ

| 項目 | 内容 |
|---|---|
| keytar ネイティブモジュール | electron-builder でリビルドが必要 (`electron-rebuild` または `postinstall` フック) |
| コード署名なし | Windows: SmartScreen 警告、macOS: Gatekeeper 警告が出る。「詳細」→「実行」で回避可能 |
| JAR 内 JAR の再帰 | 深さ 2 までに制限。無限ループ防止のため |
| Gemini Rate Limit | 無料枠では 2 RPM 程度のため、デフォルトの `sleepInterval=400ms` と `parallelism=1` が推奨 |
| パッケージサイズ | Electron バンドルで約 150MB。インストーラーを経由するため許容範囲 |
